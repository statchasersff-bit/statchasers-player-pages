Why this works

3) Fix headshots (root cause + robust solution)

Headshots “not showing” is almost always one of these:

Bad URL / missing https:

CORS / hotlink blocked

Mixed content (http inside https)

Image field missing for some players

Broken caching or rate-limited CDN

Best practice: always have a fallback chain

Try order:

Your preferred headshot URL

Sleeper headshot (if you use Sleeper IDs)

ESPN headshot (if you store ESPN IDs)

Local placeholder silhouette with team color

In the UI, never show “broken image” — show a clean placeholder.

Example in HTML:

<img
  src="${primaryHeadshot}"
  onerror="this.onerror=null; this.src='${fallbackHeadshot}'"
  alt="${playerName}"
/>

If you tell me what field your dataset uses for headshots (or if you’re using Sleeper IDs), I can give you the exact final URL pattern + mapping.
