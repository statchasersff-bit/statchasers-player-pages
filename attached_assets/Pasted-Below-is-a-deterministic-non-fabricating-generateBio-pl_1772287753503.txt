Below is a deterministic, non-fabricating generateBio(player) logic flow you can hand to Replit AI. It assumes you already have:

player from Sleeper (name/team/pos/age/height/weight/exp, etc.)

draftMap[player_id or slug] (draft year/round/pick/team, college)

seasonStats[player_id] (per-season fantasy + games played + finishes if you compute)

transactions/teamHistory[player_id] (optional, if you track team changes)

injuryLog[player_id] (optional; only player-specific, and only if credible)

1) Output shape (what generateBio returns)
type BioPayload = {
  snapshot_bullets: string[];
  style: {
    how_he_wins: string;
    how_he_wins_tags: string[];
    fantasy_translation: string;
    fantasy_translation_tags: string[];
  };
  timeline: { label: string; badge?: string; text: string }[];
  career_context_tiles: { title: string; text: string }[];
  narrative_paragraphs: string[];
  last_updated: string;              // ISO date
  sources: { label: string; url: string }[];
  debug?: Record<string, any>;       // optional for QA
};
2) Data normalization (must happen first)
2.1 Normalize identity fields

pos = normalizePosition(player.position) (QB/RB/WR/TE)

team = normalizeTeam(player.team) (BUF, etc.)

name = player.full_name

age = player.age ?? null

height = player.height ?? null (convert to 6'5" style only in UI)

weight = player.weight ?? null

exp = player.years_exp ?? null

2.2 Merge enrichment

draft = draftMap[player.id] || draftMap[player.slug] || null

college = draft?.college || player.college || null

seasons = seasonStats[player.id] || [] (sorted ascending by year)

Each season row ideally includes:

year

games

ppg_ppr (or your canonical scoring)

total_points_ppr

pos_finish (QB3, WR14, etc.) if you compute

teamHistory = ... (optional)

injuries = ... (optional, and only include if sourced + relevant)

2.3 Derived signals (objective only)

Compute these from your own stats:

rookieYear = draft?.year ?? firstSeasonYear ?? null

lastSeason = max(seasons.year) or null

bestSeason = argmax(seasons, ppg_ppr) (or total points)

breakoutSeason (rules below)

trendRecent using last 2–3 seasons:

ppg_last = seasons[last].ppg

ppg_prev = seasons[last-1].ppg

trend = ppg_last - ppg_prev

durability:

games_last_season, avg_games_last3

Breakout rules (deterministic):
Use position-specific thresholds on PPG (in your default scoring, ideally PPR for WR/RB/TE; QB standard is fine).

QB breakout if ppg >= 19 (or top-12 finish if you have ranks)

RB breakout if ppg >= 14

WR breakout if ppg >= 13

TE breakout if ppg >= 10.5
Pick the first season meeting threshold and at least games >= 8. If none, breakoutSeason = null.

These are “good enough” defaults—tune later, but keep deterministic.

3) Build Section A: Snapshot Bullets

Start empty bullets = []. Add up to 6 bullets, in this priority order:

Draft bullet (only if draft exists)

"Drafted ${draft.year}: Round ${draft.round}, Pick ${draft.pick} (${draft.team})"

Entry/early role bullet

If rookieYear exists:

"Entered the league in ${rookieYear} and developed into a ${roleDescriptor(pos, seasons)}."

roleDescriptor rules:

if last season games >= 10 and last season ppg above breakout threshold -> “high-end fantasy starter”

else if games >= 10 -> “weekly fantasy option”

else -> “depth option”

Breakout bullet (if breakoutSeason exists)

"Broke out in ${breakoutSeason.year}, establishing ${pos === 'QB' ? 'QB1-level' : 'starter-level'} fantasy production."

Peak bullet (if bestSeason exists and it’s not the same as breakout season)

If you have pos_finish: "Career peak: ${bestSeason.year} (${pos}${bestSeason.pos_finish})."

Else: "Career peak: ${bestSeason.year} (${bestSeason.ppg.toFixed(1)} PPG)."

Durability bullet (only if you have clean games data)

If avg_games_last3 >= 14: "Generally available year-to-year, averaging ${avg_games_last3.toFixed(1)} games over the last 3 seasons."

If avg_games_last3 <= 11: "Availability has been a factor recently, averaging ${avg_games_last3.toFixed(1)} games over the last 3 seasons."

Current role bullet

Based on last season + team status:

"Enters ${currentYear()} as the ${team} ${pos} with a clear role in the offense."

If games low or team change: "Enters ${currentYear()} with role volatility that can swing weekly fantasy outcomes."

Finally:

De-duplicate similar bullets

Cap at 6

Remove any bullet that would be “guessy” (no coach mentions, no “elite arm” claims unless your data supports it)

4) Build Section B: Style Cards (How he wins / Fantasy translation)
4.1 Tags (position-based, derived)

Create tags from objective signals (not vibes). Examples:

QB tags

If rushing share high (you can compute rush yards / total yards, or rush attempts/game):

add "Dual-threat", "Rushing floor"

If passing ppg stable (abs(trend) small over 2 seasons):

add "Stable weekly role"

If TD rate volatile (you already have TD regression logic):

add "TD volatility"

RB tags

If carries/game high: "Volume runner"

If targets/game high: "Pass-game role"

If goal-line carries share high: "Goal-line equity"

If committee (snap% low but points ok): "Efficiency-driven"

WR tags

If targets/game high: "Target earner"

If aDOT high: "Downfield role"

If YAC heavy (if you have): "YAC-driven"

If boom-bust (high weekly variance): "Spike-week profile"

TE tags

If targets high: "Volume TE"

If TD heavy: "Red-zone role"

If low targets, high TD: "TD-dependent"

4.2 “How he wins” paragraph (template, position-specific)

Keep it generic but true.

QB

If dual-threat: "Creates production with both arm and legs, extending plays and adding rushing value when passing efficiency dips."

Else: "Wins with pocket passing and volume-driven production when the offense is operating efficiently."

RB

If pass-game role: "Produces through a blend of rushing volume and receiving usage, keeping weekly output stable across game scripts."

Else: "Primarily a rushing-based producer whose weekly output is tied to carries and touchdown opportunities."

WR

If target earner: "Functions as a primary read in the passing game, stacking targets and converting volume into consistent weekly production."

Else if downfield role: "Operates as an explosive, downfield option—lower target volume but high leverage looks that can spike weeks."

Else: "Profiles as a complementary receiver whose fantasy output swings with weekly usage and efficiency."

TE

If red-zone role: "Provides fantasy value through high-leverage red-zone usage, with weekly output often tied to touchdowns."

Else: "Wins through steady involvement underneath, relying on target volume more than splash plays."

4.3 Fantasy translation paragraph (template)

Tie to floor/ceiling drivers using your computed signals.

Examples:

Dual-threat QB: "Weekly ceiling is driven by multi-touchdown games, while rushing attempts stabilize floor even in quieter passing weeks."

Target earner WR: "Floor comes from consistent target volume; ceiling spikes when deep targets and red-zone looks converge."

TD-dependent TE: "Floor can be thin without a touchdown, but red-zone usage creates spike-week potential."

Return tags arrays capped at 4.

5) Build Section C: Timeline (4–8 entries)

Start timeline = []. Add entries in this order:

Draft entry (if draft)

{ label: String(draft.year), badge: "Drafted", text: "Selected Round X, Pick Y by TEAM out of COLLEGE." }

Rookie/early role (if rookieYear and you have season data)

Label rookieYear

Badge: "Early career"

Text: "Entered the league and began carving out a role." (keep generic)

Breakout entry (if breakoutSeason)

Badge: "Breakout"

Text references breakout year and PPG or finish (if available)

Peak entry (if bestSeason)

Badge: "Peak"

Text: "Career-high production..."

Recent trend (if last 2 seasons exist)

If trend >= +2.0: badge "Uptrend"

If trend <= -2.0: badge "Downtrend"

Else badge "Stable"

Text: "Recent production has been ..."

Injury entry (only if you have a real injury log and it mattered)

Badge "Injury"

Text: "Missed time due to ..." (no medical detail beyond public designation)

Finally:

De-dupe years

Cap at 8

If you have fewer than 3 entries, hide timeline section in UI

6) Build Section D: Career Context Tiles (4 tiles)

These are short labels (no tables). Compute using your signals:

Tiles (always attempt these 4):

Floor driver

QB: "Rushing usage" if dual-threat else "Passing volume/efficiency"

RB: "Receiving work" if targets high else "Carry volume"

WR: "Target volume" if targets high else "High-leverage targets"

TE: "Targets" if targets high else "Red-zone usage"

Ceiling driver

QB: "Multi-TD games + rushing TDs"

RB: "Goal-line equity + big plays"

WR: "Deep targets + TDs"

TE: "Red-zone TDs"

Stability

Based on variance / games played consistency:

"Stable" / "Moderate" / "Volatile"

Rule example: if weekly stddev in last season is top 25% -> Volatile

Risk note
Pick one (deterministic):

If avg_games_last3 low: "Availability"

Else if role metrics declining (OppΔ negative): "Role decline risk"

Else if TD dependency high: "TD dependency"

Else "Scheme/usage swings"

7) Build Section E: Narrative Paragraphs (2–4)

This is the “PlayerProfiler vibe” but generated from facts.

Paragraph 1 (origin)

Template:

If draft exists:
"${name} entered the NFL as a ${draft.round}${ordinal(draft.round)}-round pick (${draft.pick} overall) in ${draft.year} out of ${college || 'college'}."

Else:
"${name} entered the NFL and quickly began establishing a role at ${pos}."

Add one more sentence using rookieYear/early seasons:

"By ${rookieYear + 1} he was seeing meaningful snaps, setting the foundation for his fantasy profile." (only if you have seasons)

Paragraph 2 (development + breakout/peak)

If breakoutSeason:
"The turning point came in ${breakoutSeason.year}, when his production climbed into ${pos}-starter territory (${ppg} PPG).".

If bestSeason and not same year:
"His peak arrived in ${bestSeason.year}, delivering his best combination of volume and fantasy output."

Paragraph 3 (current fantasy summary)

Use your “Fantasy translation” language:
"For fantasy managers, his floor is driven by ${floorDriver}, while his ceiling spikes when ${ceilingDriver} hit in the same week."

Optional Paragraph 4 (risk/stability)

Only include if you have a meaningful risk signal:

durability low

role declining

extreme TD dependency
Keep it neutral:

"The main variable going forward is ${riskNote}, which can create week-to-week variance even when the overall role remains intact."

Hard rules (to prevent BS)

No coach names

No “elite arm / generational / best in league”

No awards unless you have a verified awards dataset

No “improved mechanics” type claims

8) Sources + last_updated

Always include:

Team bio URL pattern (if you have it)

NFL profile URL pattern (if you have it)

Pro-Football-Reference (generic player search link is fine, or omit if you can’t generate safely)

Example:

sources = [
  { label: "Sleeper", url: "https://sleeper.com" },
  { label: "NFL", url: "https://www.nfl.com/players/" },
  { label: "Pro-Football-Reference", url: "https://www.pro-football-reference.com" }
];

last_updated = new Date().toISOString().slice(0,10)

9) Full pseudo-code (copy to Replit AI)
export function generateBio(player, ctx): BioPayload {
  const { draftMap, seasonStatsMap, injuryMap } = ctx;

  const pos = normalizePosition(player.position);
  const team = normalizeTeam(player.team);
  const name = player.full_name;

  const draft = draftMap[player.player_id] || draftMap[player.slug] || null;
  const seasons = (seasonStatsMap[player.player_id] || []).slice().sort((a,b)=>a.year-b.year);

  const derived = computeDerived(pos, seasons, draft);

  const snapshot_bullets = buildSnapshotBullets({ name, pos, team, draft, seasons, derived });

  const style = buildStyle({ pos, seasons, derived });

  const timeline = buildTimeline({ pos, draft, seasons, derived, injuries: injuryMap?.[player.player_id] });

  const career_context_tiles = buildCareerContextTiles({ pos, seasons, derived });

  const narrative_paragraphs = buildNarrative({ name, pos, team, draft, seasons, derived, style, tiles: career_context_tiles });

  return {
    snapshot_bullets,
    style,
    timeline,
    career_context_tiles,
    narrative_paragraphs,
    last_updated: new Date().toISOString().slice(0, 10),
    sources: defaultSources(),
    debug: {
      breakoutYear: derived.breakoutSeason?.year ?? null,
      bestYear: derived.bestSeason?.year ?? null,
      trend: derived.trend ?? null
    }
  };
}
